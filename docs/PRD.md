# 주읽이 (StockRead) — PRD

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | 주읽이 (StockRead) |
| **한 줄 정의** | 주린이를 위한 AI 기반 투자 해석 서비스 |
| **타겟 유저** | 주식 초보 투자자 (20~40대, 투자 시작 1년 이내) |
| **핵심 가치** | 뉴스/차트/재무를 판단 가능한 언어로 해석 |
| **MVP 채널** | Web(랜딩) + Telegram Bot(핵심) + YouTube Shorts(유입) |
| **수익 모델** | 초기 무료 → 검증 후 프리미엄 구독 (₩4,900~9,900/월) |
| **차별점** | 데이터 제공이 아닌 투자 해석 제공. 성향별 맞춤 해석 |

---

## 2. MVP 기능 목록

| # | 기능 | 채널 |
|---|------|------|
| 1 | 랜딩 페이지 (소개 + 리포트 예시 + 텔레그램 유도) | Web |
| 2 | 봇 온보딩 (성향 + 종목 구독) | Telegram |
| 3 | 종목/성향 설정 변경 (/edit) | Telegram |
| 4 | 데이터 수집 파이프라인 | Backend |
| 5 | AI 종목 리포트 생성 (성향별) | Backend |
| 6 | 일일 리포트 자동 발송 (07:00 KST) | Telegram |
| 7 | 쇼츠 스크립트 + MP4 자동 생성 | Backend |
| 8 | MP4 검토용 전송 (09:00 KST) → 운영자 수동 업로드 | Telegram |

## 3. 웹 랜딩 페이지

### 역할
서비스 입구. 소개 + 텔레그램 봇 유도.

### 섹션 구성
| 섹션 | 내용 |
|------|------|
| 히어로 | 후킹 카피 + "텔레그램에서 시작하기" CTA. 임팩트 있는 비주얼 |
| 리포트 예시 | **인터랙티브** — 종목 탭으로 골라서 볼 수 있음 (NVDA / TSLA / 삼성전자 등) |
| 특징 3가지 | ① AI 해석 ② 투자 성향 맞춤 ③ 매일 아침 자동 |
| 하단 CTA | "무료로 시작하기" → t.me/StockReadBot |

### 안 만드는 것
- ❌ 회원가입 / 로그인
- ❌ 종목 설정
- ❌ 대시보드

### 디자인 방향
- 프리미엄 + 깔끔
- 후킹 되는 예쁜 UI
- 모바일 퍼스트 (유튜브 쇼츠 → 모바일 유입이 대부분)

### 기술
- Next.js 정적 페이지 1장
- 반응형

## 4. 텔레그램 봇

### 역할
서비스 핵심 허브. 온보딩 + 설정 + 리포트 수신.

### 4-1. 온보딩 플로우

```
유저: /start

봇: 안녕하세요! 📊 주읽이입니다.
    매일 아침 7시, AI가 해석한 종목 리포트를 보내드려요.
    
    먼저 투자 성향을 선택해주세요:
    [🛡 안정형] [⚖️ 중립형] [🔥 공격형]

유저: [중립형] 클릭

봇: ⚖️ 중립형으로 설정했어요!
    
    관심 종목을 선택해주세요 (최대 3개):
    
    🇺🇸 미국
    [AAPL] [NVDA] [TSLA] [MSFT] [SPY] [QQQ]
    
    🇰🇷 한국
    [삼성전자] [SK하이닉스] [NAVER] [카카오] [LG에너지] [현대차]

유저: NVDA, 삼성전자, TSLA 선택

봇: ✅ 설정 완료!
    
    📊 구독 종목: NVDA, 삼성전자, TSLA
    ⚖️ 투자 성향: 중립형
    ⏰ 리포트: 매일 오전 7시
    
    내일 아침부터 리포트를 보내드릴게요!
```

### 4-2. 일일 리포트 형식

종목별 개별 메시지로 발송 (3종목 = 3개 메시지)

```
📊 2026.03.10 주읽이 리포트

━━━━━━━━━━━━━━━
💚 NVDA (엔비디아)
━━━━━━━━━━━━━━━

1️⃣ 현재 위치
• 종가: $875.40 (+2.3%)
• 52주 고점 대비: 91%

2️⃣ 기술적 상태
• RSI 72 → 과열 구간 근접
• 20일 이평선 위 → 단기 상승 추세

3️⃣ 최근 이슈
• AI 서버 수요 증가 뉴스 (호재)
• 실적 발표 2주 전

4️⃣ 해석 (⚖️ 중립형 기준)
단기 상승 이후 조정 가능성 존재.
분할 접근이 적절한 구간.
실적 발표 전 변동성 확대 가능.

⚠️ 본 리포트는 투자 참고 정보이며
투자 판단의 책임은 본인에게 있습니다.
```

### 4-3. 봇 명령어

| 명령 | 동작 |
|------|------|
| `/start` | 온보딩 |
| `/my` | 현재 설정 확인 (성향 + 구독 종목) |
| `/edit` | 성향 + 종목 수정 (온보딩 버튼 뷰 재표시) |
| `/help` | 명령어 안내 |

### 4-4. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 종목 3개 꽉 찬 상태에서 추가 시도 | "최대 3개까지 가능해요. 먼저 다른 종목을 해제해주세요" |
| 이미 구독 중인 종목 선택 | 토글 해제 (선택/해제 방식) |
| 지원하지 않는 종목 | 버튼 기반이라 해당 없음 |
| 온보딩 안 한 상태에서 `/my` | 온보딩 먼저 유도 |

## 5. AI 리포트 생성

### 입력 데이터

| 데이터 | 출처 | 용도 |
|--------|------|------|
| 종가, 전일비 | FMP / KRX | 현재 위치 |
| 52주 고저 | FMP / KRX | 고점 대비 % |
| RSI, 이평선 | FMP / 계산 | 기술적 상태 |
| 최근 뉴스 | FMP / DART | 이슈 요약 |
| 실적 일정 | FMP | 이벤트 안내 |

### LLM 프롬프트 구조

```
너는 주식 초보를 위한 투자 해석 AI야.

[데이터]
종목: {ticker}
종가: {price} ({change})
52주 고점 대비: {pct_from_high}
RSI: {rsi}
이평선 위치: {ma_position}
최근 뉴스: {news}
실적 발표: {earnings_date}

[유저 성향]
{risk_type}

[규칙]
- 매수/매도 추천 금지
- "리스크 높음", "변동성 가능성" 등 표현 사용
- 초보자가 이해할 수 있는 쉬운 말
- 4개 섹션: 현재 위치 / 기술적 상태 / 최근 이슈 / 해석
```

### 성향별 해석 톤

| 성향 | 해석 톤 |
|------|---------|
| 🛡 안정형 | 보수적. 리스크 강조, 분할/관망 권유 |
| ⚖️ 중립형 | 균형. 양면 분석, 조건부 접근 |
| 🔥 공격형 | 적극적. 기회 강조, 단기 모멘텀 언급 |

## 6. 쇼츠 자동 생성

### 스크립트 JSON 구조

```json
{
  "date": "2026-03-10",
  "title": "3월 10일 미국·한국 증시 핵심 요약",
  "market_summary": "미국 반도체 강세, 한국 2차전지 약세",
  "highlights": [
    "엔비디아 AI 수요 호재로 +2.3%",
    "테슬라 기술적 반등 시도",
    "삼성전자 외국인 순매도 지속"
  ],
  "interpretation": "단기 과열 구간 접근, 신규 진입 신중",
  "tts_script": "오늘 미국 증시는 반도체 강세로..."
}
```

### 영상 스펙

| 항목 | 값 |
|------|-----|
| 길이 | 30~60초 (시황에 따라 유동적) |
| 비율 | 세로 9:16 |
| 비주얼 | 다크 배경 + 네온 텍스트 (금융 느낌) |
| TTS | OpenAI TTS, 남성, 차분한 뉴스 톤 |
| CTA | 영상 내 없음 (유튜브 프로필/설명란에만) |

### 영상 구조

| 시간 | 내용 |
|------|------|
| 0~3초 | 강한 제목 (후킹) |
| 3~10초 | 오늘 시장 요약 |
| 10~20초 | 핵심 종목 2~3개 |
| 20~30/60초 | 시장 해석 |

### 생성 파이프라인

```
LLM → 스크립트 JSON
  ↓
OpenAI TTS → 음성 파일 (mp3)
  ↓
Remotion → 텍스트 애니메이션 + 차트 이미지 + 음성 합성
  ↓
MP4 출력
```

## 8. 데이터 모델

```sql
-- 사용자
users (
  id            SERIAL PRIMARY KEY,
  telegram_id   TEXT UNIQUE NOT NULL,
  risk_type     TEXT NOT NULL,  -- 안정/중립/공격
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
)

-- 구독
subscriptions (
  id            SERIAL PRIMARY KEY,
  user_id       INT REFERENCES users(id),
  ticker        TEXT NOT NULL,
  market        TEXT NOT NULL,  -- US / KR
  created_at    TIMESTAMPTZ DEFAULT NOW()
)

-- 지원 종목
stocks (
  ticker        TEXT PRIMARY KEY,
  name_ko       TEXT,
  name_en       TEXT,
  market        TEXT NOT NULL,  -- US / KR
  is_active     BOOLEAN DEFAULT TRUE
)

-- 일일 리포트
daily_reports (
  id            SERIAL PRIMARY KEY,
  ticker        TEXT NOT NULL,
  date          DATE NOT NULL,
  report_json   JSONB,
  status        TEXT DEFAULT 'generated',  -- generated / sent
  created_at    TIMESTAMPTZ DEFAULT NOW()
)

-- 쇼츠 콘텐츠
shorts_content (
  id            SERIAL PRIMARY KEY,
  date          DATE NOT NULL,
  script_json   JSONB,
  mp4_path      TEXT,
  status        TEXT DEFAULT 'generated',  -- generated / sent / uploaded
  created_at    TIMESTAMPTZ DEFAULT NOW()
)
```

## 부록: 톤 & 보이스 가이드

### 원칙
- **존댓말** 사용 (해요체)
- **친근하고 쉬운 말** — 주린이가 친구한테 설명 듣는 느낌
- **딱딱한 전문 용어 피하기** — 쓸 때는 바로 풀어서 설명

### 톤 예시
| ❌ 딱딱 | ✅ 친근 |
|---------|---------|
| 단기 상승 이후 조정 가능성 존재 | 최근 많이 올랐어요. 조정이 올 수도 있어요 |
| 분할 접근이 적절한 구간 | 한 번에 사기보다 나눠서 접근하는 게 좋아 보여요 |
| 과열 구간 근접 | 지금 좀 과열된 상태예요. 조심하세요! |
| 변동성 확대 가능 | 가격이 많이 흔들릴 수 있어요 |
| RSI 72 → 과열 구간 | RSI가 72예요. 보통 70 넘으면 과열 신호로 봐요 |
| 52주 고점 대비 91% | 1년 중 가장 높았던 가격의 91% 위치에 있어요 |

### 리포트 예시 (변경 후)
```
📊 2026.03.10 주읽이 리포트

━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━

2️⃣ 차트가 말해주는 것
• RSI가 72예요. 70 넘으면 과열 신호로 봐요

3️⃣ 무슨 일이 있었나요?
• AI 서버 수요가 늘고 있다는 뉴스가 나왔어요 (호재 📈)
• 실적 발표가 2주 뒤에 있어요

4️⃣ 이렇게 보시면 돼요 (⚖️ 중립형)
최근 많이 올랐기 때문에 조정이 올 수도 있어요.
한 번에 사기보다 나눠서 접근하는 게 좋아 보여요.
실적 발표 전에는 가격이 흔들릴 수 있으니 참고하세요!

⚠️ 이 리포트는 참고용이에요.
```
